Swift Search

Privacy Policy

Last updated: May 8, 2026

Swift Search does not collect, store, or transmit any personal data to any server owned or operated by Swift Search. Everything stays on your device.

What Swift Search does

Swift Search is a Chrome extension that allows you to highlight text on any webpage and instantly search it across your chosen websites. It stores your settings and website list locally using Chrome's built-in storage.

Data we do not collect

Swift Search does not collect, store, or have access to:

  • Your browsing history or visited URLs
  • The text you highlight or search
  • Your name, email address, or any identifying information
  • Any data from the websites you visit
  • Search queries or results

What is stored locally

Swift Search stores the following data on your device only, using Chrome's storage APIs:

  • Your list of saved websites and their order
  • Your preferences (dark mode, display mode, open-in-current-tab, search suggestions)
  • Site groups you have created
  • Your search history — the queries you have typed into the Swift Search bar, stored locally to power the search suggestions dropdown. Individual history items can be deleted at any time from within the extension. This data is never transmitted anywhere.
  • Your personal usage stats (total searches, searches today, daily streak, personal best day, searches per site, last used site) — used only to display your stats in the popup and never transmitted anywhere
  • Saved text excerpts from the "Save for Later" feature (including the highlighted text, source URL, Text Fragment URL, timestamp, word count, and any notes you add) — stored locally and never uploaded to any server
  • Whether you have seen the onboarding message

If Chrome Sync is enabled on your account, Chrome may sync this data across your own devices. This sync is handled entirely by Google — Swift Search has no involvement in or access to this process.

Export and Share features

Swift Search includes features to export saved items and share highlighted text:

  • Export to Word — generates a document file (.doc) on your device containing your selected saved items. The file is created entirely in your browser and never uploaded anywhere.
  • Export to Google Docs — copies formatted text to your clipboard and opens a new Google Doc. The data is not sent to Swift Search servers; you manually paste it into Google Docs, which is governed by Google's privacy policy.
  • Share — copies highlighted text and a source URL to your clipboard for sharing via messaging apps or social media. No data is transmitted by Swift Search; sharing is handled by the apps you paste into.

Third-party network requests

Swift Search makes the following outbound network requests, none of which transmit personal data:

  • Google Favicon Service — used to load website icons in the search menu. The domain name of the website is sent to fetch its icon. No personal data is included.
  • DuckDuckGo Favicon Service (icons.duckduckgo.com) — used as a fallback favicon source. Same as above.
  • Dictionary API (api.dictionaryapi.dev) — called when you use the Define button in the floating menu. The word you are looking up is sent to the API to retrieve its definition. No other data is included.
  • DuckDuckGo Instant Answer API (api.duckduckgo.com) — called when you use the Brief button in the floating menu. The highlighted text is sent to retrieve a short summary. No personal data is included.
  • Google Search Suggestions (suggestqueries.google.com) — called when the search suggestions feature is enabled in Settings. The text you type in the search bar is sent to retrieve autocomplete suggestions. This feature is off by default and must be enabled manually.
  • Google Translate — when you use the Translate button in the floating menu, your browser opens Google Translate in a new tab with the highlighted text in the URL. This is standard web browsing and is governed by Google's privacy policy.
  • Website homepages — when you add a new website, Swift Search may fetch its homepage to automatically detect how to search it. This is a standard HTTP request containing no personal data.
  • Search websites — when you perform a search, your browser opens the relevant website in a new tab with your search query in the URL. This is standard web browsing and is governed by the privacy policy of the website you are searching.

Permissions

Swift Search requests the following Chrome permissions, each used solely to deliver its core functionality:

  • storage — to save your websites, preferences, and settings locally
  • tabs — to open search results in a new tab and detect the active tab when using the keyboard shortcut
  • scripting — to inject the floating search menu into web pages
  • alarms — to keep the extension's background service worker alive so searches feel instant
  • host permissions (<all_urls>) — required so the floating search menu can appear on any webpage you visit

Children's privacy

Swift Search does not knowingly collect any data from anyone, including children under the age of 13. As no data is collected at all, the extension is safe for use by all ages.

Changes to this policy
